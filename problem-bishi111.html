·这道题在问什么

给你一个长度为 n 的数组，然后有 m 次"操作"，每次操作给三个数 l, r, k，意思是"把下标从 l 到 r 的这一整段元素，每个都加上 k"。m 次操作都做完之后，输出整个数组最终的样子。

题目原文（换成大白话）：数组 a 长度 n；给定 (l, r, k)，要执行 a[l] += k, a[l+1] += k, ..., a[r] += k；重复 m 次操作后，输出完整数组。

·照着样例读一遍输入输出

输入

3 2          // n=3 个数，m=2 次操作
1 2 3        // a[1]=1, a[2]=2, a[3]=3
1 2 4        // 操作1：给 [1,2] 都加 4
3 3 -2       // 操作2：给 [3,3] 都加 -2

输出

5 6 1        // 见下方逐步推导

手动验证一下：初始 [1, 2, 3]。操作1 给 [1,2] 加 4 → [5, 6, 3]。操作2 给 [3,3] 加 -2 → [5, 6, 1]。和输出对上了。

·为什么不能"暴力"，需要差分

暴力做法：每次操作都老实地把 [l,r] 里每个数都加一遍，单次操作最坏 O(n)。m 次操作下来是 O(n·m)，这题 n, m ≤ 10^5，最坏 10^5 × 10^5 = 10^10 次运算，1 秒的时间限制根本跑不完，会 TLE。

差分数组的思路：维护一个"差分"数组 diff[i] = a[i] - a[i-1]（相邻两个数的差）。这样一来，"把 [l,r] 整体加 k"这个操作，只需要改 diff 的两个位置：diff[l] += k，diff[r+1] -= k——不管区间多长，都是 O(1)。所有操作做完后，对 diff 求一次前缀和，就能一次性还原出最终数组。这正是 Day6「差分数组」讲的技巧，这里是它的完整模板题。

·容易踩的坑

1. 数值范围：还是要开 long long

a[i] 和 k 都在 ±10^9 量级，m 最多 10^5 次操作叠加，最坏情况下单个位置能累加到 10^5 × 10^9 = 10^14，早就超过 int 的范围（约 ±2.1×10^9）。a 数组、diff 数组、累加变量、最终结果，统统要用 long long。

2. diff 数组要多开一位

当 r = n 时，diff[r+1] 就是 diff[n+1]——如果 diff 数组只开到下标 n，这里就越界了。把 diff 开成 n+2 大小（下标 0..n+1），多出来的 diff[n+1] 只是写进去但从来不会在还原时被读到，安全又省心。

3. 差分数组本身不是答案，别忘了最后一步"求前缀和"

diff 只是一个记录"变化量"的中间结构。做完所有操作后，一定要对 diff 求一次前缀和（acc += diff[i]）才能得到真正的最终数组——这一步很容易在写代码时漏掉。

第 1 行两个整数 n, m
第 2 行n 个整数，初始数组
接下来 m 行每行三个整数 l, r, k，代表一次区间加操作
输出一行 n 个整数，全部操作完成后的数组

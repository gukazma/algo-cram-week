·这道题在问什么

给你一个长度为 n 的数组，然后有 q 次询问，每次问"区间 [l, r] 里所有数加起来是多少"。数据范围 n, q 最大到 10^6（一百万），这个规模决定了做法——见下面"为什么不能暴力"。

·没做过 ACM/OJ？先搞懂输入输出是怎么回事

力扣（LeetCode）是让你补全一个函数；而 ACM/OJ（牛客、洛谷、Codeforces 这类）要求你写一个完整的、独立运行的程序：程序从"标准输入"读数据（C++ 里用 cin>> 或 scanf），把答案打印到"标准输出"（cout<< 或 printf）。评测机把"输入描述"的内容喂给你的程序，再逐字比对你的输出和标准答案，一个字符都不能错。

照着本题的样例，输入输出每一行代表什么，对照着看一遍：

输入（你的程序读到的）

3 2          // n=3 个数，q=2 次查询
1 2 4        // a[1]=1, a[2]=2, a[3]=4
1 2          // 第 1 次查询：l=1, r=2
2 3          // 第 2 次查询：l=2, r=3

输出（你的程序要打印的）

3            // [1,2]：a1+a2 = 1+2
6            // [2,3]：a2+a3 = 2+4

题目的 l, r 是从 1 开始编号的（不是 0），下面的代码和动画都保持这个编号方式，省得来回换算。

·为什么不能"暴力"

暴力做法：每次查询 [l,r] 就从 l 加到 r，单次 O(n)。q 次查询下来是 O(n·q)，最坏 10^6 × 10^6 = 10^12 次运算。现代 CPU 一秒大概能跑 10^8~10^9 次，10^12 次要跑几分钟到几小时——而 OJ 通常只给 1~2 秒，必然超时（TLE, Time Limit Exceeded）。

需要一种"预处理一次 O(n)，之后每次查询 O(1)"的办法——这就是前缀和：提前算出 prefix[i] = 前 i 个数的和，查询 [l,r] 直接用 prefix[r] - prefix[l-1]，不用重新累加。

·容易踩的坑

1. 忘记开 long long，答案溢出

a[i] 绝对值最大到 10^9，最多 10^6 个数，前缀和最大能到 10^9 × 10^6 = 10^15。int 最大只能表示约 2.1×10^9，装不下，必须用 long long（C++）存 a、prefix 和答案，否则会算出错误的负数或乱码一样的结果。

2. cin/cout 不加速，n、q 到 10^6 级别可能超时

C++ 的 cin/cout 默认要和 C 的 stdio 保持同步，速度较慢。数据量大时建议要么用 scanf/printf，要么在 main 开头加 ios_base::sync_with_stdio(false); cin.tie(nullptr); 再用 cin/cout。

3. 下标从哪里开始，全程要一致

题目里 l, r ∈ [1, n]，是从 1 开始编号的。把数组开成 n+1 大小、下标 1 到 n 存数据（下标 0 空着不用），可以让代码里的下标和题目描述里的编号完全对上，不用再心算"要不要减一"。

a 和 prefix 两个数组开 n+1，原因其实不一样：

  • prefix 的 n+1 是必须的：公式 prefix[r] - prefix[l-1] 在 l=1 时要用到 prefix[0]（"前 0 个数的和" = 0），少了它这个哨兵，l=1 的查询就会越界访问 prefix[-1]。
  • a 的 n+1 只是图方便：如果开成 vector<long long> a(n)（下标 0..n-1），代码里每次读写都要写成 a[i-1] 才能对应题目里的 a[i]，容易漏减或多减；开成 n+1、下标 1..n、把下标 0 空着不用，就能让代码里的 a[i] 和题目里的 a[i] 完全一致，浪费一个格子换来全程不用心算偏移量。
vector<long long> a(n);        // 0-indexed: a[0]..a[n-1]
scanf("%lld", &a[i - 1]);      // 每次都要减一，容易出错

vector<long long> a(n + 1);    // 1-indexed: a[0] 空着，a[1]..a[n] 存数据
scanf("%lld", &a[i]);          // 代码里的 a[i] 就是题目里的 a[i]
输入/输出格式
第 1 行两个整数 n, q
第 2 行n 个整数，数组内容
接下来 q 行每行两个整数 l, r，代表一次查询
输出对每次查询输出一行，一个整数（区间和）
